Telco Customer Churn Prediction

This project focuses on analysing customer behaviour in a telecom company and predicting whether a customer is likely to churn (leave the service).

The goal was not just to build a machine learning model, but also to understand the factors influencing churn and translate those insights into actionable business decisions.

🧠 Key Objectives
Understand the dataset and business problem
Perform data cleaning and preprocessing
Explore patterns using EDA
Build a simple neural network model
Evaluate model performance
Derive insights and suggest business actions
⚙️ Approach
Cleaned and preprocessed the dataset (handling missing values, encoding, scaling)
Performed exploratory data analysis to identify churn patterns
Built a feedforward neural network using TensorFlow/Keras
Evaluated performance using confusion matrix and classification metrics
Interpreted results in a business context
🔍 Key Insights
Customers with month-to-month contracts are more likely to churn
Higher monthly charges are associated with higher churn
Customers using fewer additional services tend to leave more
💡 Business Recommendations
Encourage customers to switch to long-term plans
Promote additional services to increase engagement
Identify high-risk customers for targeted retention strategies
🛠️ Tech Stack
Python
Pandas, NumPy
Seaborn, Matplotlib
Scikit-learn
TensorFlow / Keras
